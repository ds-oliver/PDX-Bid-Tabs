# Data Model Overview

This project is an internal bid lookup tool: it parses Bid Tabulation Excel sheets into a clean analytical extract for historical analysis. It does not report selection outcomes.

## Production Dashboard Model (Current)
The production semantic model for Power BI is generated by `scripts/build_star_schema.py` and stored in `data_out/`:
- `Projects.csv`
- `Items.csv`
- `Bids.csv`

## Relationship Pattern (Star-Like)
- `projects[project_id]` 1:* `items[project_id]`
- `items[item_id]` 1:* `bids[item_id]`
- `Bids` is the primary fact table.

## Staging/Lineage Extract
- `compiled_excel_itemized_clean.csv` remains the canonical staging extract at bidder-line grain (Project x Schedule x Line Item x Bidder), including totals rows.

## Dashboard Use (ODOT-Style Reference)
The end dashboard is designed for bid analytics similar to the ODOT example. These are the intended uses:
- Project context and slicers: `Projects`
- Item context and drilldown: `Items`
- Contractor pricing, ranking, and winner analytics: `Bids`
